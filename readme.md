# Análise de Sistema - Engenharia Reversa

Este repositório contém uma atividade acadêmica voltada para análise de um sistema real, utilizando conceitos de engenharia reversa, arquitetura de software e modelagem.

## Objetivo

O objetivo da atividade foi observar o funcionamento de um sistema real (site de pedidos online) e, a partir disso, identificar suas características, organização e possíveis melhorias.


## O que foi feito

Durante o desenvolvimento da atividade, foram realizadas as seguintes etapas:

- Análise do funcionamento do sistema
- Identificação das principais funcionalidades
- Avaliação da arquitetura utilizada
- Comparação com um sistema didático
- Proposta de melhorias
- Criação de um diagrama UML simples


## Estrutura do Projeto

Dentro da pasta `analise-sistema` estão os arquivos da atividade:

- `analise.md` → análise do sistema real  
- `comparacao.md` → comparação com sistema didático  
- `arquitetura.md` → proposta de arquitetura  
- `melhorias.md` → problemas e melhorias  
- `modelagem.png` → diagrama UML  


## Ferramentas utilizadas

- Visual Studio Code
- Git e GitHub
- Draw.io (para o diagrama)


## Diagrama UML

O diagrama foi feito de forma simples, representando as principais entidades do sistema como cliente, pedido, produto e categoria.


## Considerações finais

A atividade ajudou a entender melhor como sistemas reais funcionam, mesmo sem acesso ao código. Também foi possível perceber a importância de uma boa organização e uso de padrões para facilitar manutenção e evolução.
